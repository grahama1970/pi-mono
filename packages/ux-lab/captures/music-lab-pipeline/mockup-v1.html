📋 Implementation
What: The creative seed — a raw emotional thought from the persona that sparks the entire pipeline. This is the ONLY human input. Everything else flows from this.
Why first: Without a thought, the pipeline has no direction. The lore recall (S01) searches for THIS memory. The reference songs (S02) match THIS mood. The lyrics (S04) express THIS feeling. Every downstream stage traces back to this sentence.
Data flow: "I'm remembering the slaughter..." → /taxonomy extract → heart: [anger, sadness, fear] → This IS the emotional seed. Every downstream stage reads heart tags directly:
• S01: recall --bridges anger,sadness,fear
• S02: search --mood anger,sadness,fear
• S04: verse.emotion=fear, chorus.emotion=anger→sadness
• S05: per-phrase emotion field = heart tag for that section
Human input / persona prompt
The thought can emerge from ANY source — it does not require human input:
• /episodic-archiver — a past conversation that stuck with the persona
• /persona-journal — tonight's journal entry surfaced a buried emotion
• /consume-feed — a news article or RSS feed triggered a memory
• /dogpile — random research led to an unexpected emotional connection
• /converse — a voice conversation moment that sparks a song idea
• /memory recall — browsing old memories, one resonates strongly
• Human input — direct text from Graham or voice transcription

The pipeline can run fully autonomously (nightly: journal → thought → song) or be human-initiated.

📋 Implementation
What: Convert converged lyrics into structured JSON with per-syllable beat positions, stress marks, emotion tags, dynamics, and vocal direction per phrase.
Why here: YuE and Sonauto need TIMING, not just text. "The static hums" on beat 1 at BPM 85 means syllable "sta" starts at 1.50 and holds for 0.50 beats. Without this, the audio generator places syllables randomly. The emotion/dynamics/vocal fields drive how the model renders each phrase — whisper at pp for fear, belt at f for joy.
Data flow: lyrics_converged.md + BPM/key from S00 context → LLM annotation (via /prompt-lab managed prompt) → annotated-lyrics.json {title, bpm, key, phrases[{text, section, bar, beat, syllables[{text, beat, hold_beats, stress}], emotion, dynamics, vocal_direction}]}
⚠ NO SKILL — LLM TASK
This is pure LLM work: given lyrics + musical context → produce valid JSON matching annotated-lyrics.schema.json. The prompt must:
• Calculate beat positions from BPM and time signature
• Identify natural syllable stress patterns
• Map S00 taxonomy tags to per-phrase emotion fields
• Infer dynamics (pp→ff) from emotional arc

prompt-lab/run.sh create --name "music-lyrics-annotate" prompt-lab/run.sh eval --name "music-lyrics-annotate" --cases fixtures/whisperheads/

What: Compare generated audio against the piano-roll-spec. Score the delta. If above threshold, re-quantize the generation prompt and regenerate. Repeat until converged.
Why here: Audio generation is non-deterministic — YuE won't perfectly match the spec on first try.
